# Resource Docs

Detailed resource documentation for `devflow-config-service`.

## Purpose

This folder is the repo-local source for resource field documentation.
It should describe repo-owned models in a structured and repeatable way.

## Required section format

Each resource document should preferably include:
- `Ownership`
- `Purpose`
- `Field Table`
- `Nested Types` or `Enums`
- `Lifecycle / Status Fields` when applicable
- `Create / Update Rules`
- `Validation Notes`
- `Source Pointers`
